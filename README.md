# Web Development

This course is part of the [Computer Science program](https://www.carleton.edu/summer/pre-college/programs/cs/) of the Summer Liberal Arts Institute at Carleton College.

## Tools and frameworks used

- *Flask* for the back-end logic
- *Jinja* for templates
- *Bulma* or another framework for consistent user interface
- *SQLite3* for the permanent data storage
- *Playwright* and *pytest* for testing
- *PythonAnywhere* or *render* for hosting

## 3-week session

During the guided research part of the course students are exposed to a variety of tools and frameworks as they build a web application backed by a database.

### Course outline

1. Introduction
    - Get familiar with Python by processing data in various formats (TXT, CSV, JSON)
    - Learn basics of *Flask* routes, return types
    - Practice returning data in a format specified by the developer instead of the default chosen by *Flask*
    - Use variable rules of *Flask* routing to filter data
2. User interface
    - Get familiar with HTML and CSS
    - Use *Jinja* templates
    - Use a framework (*Bootstrap*, *Bulma*, *Pico* etc.) to have a consistent user interface
3. Interactivity
    - Use HTML forms to send requests to the back-end of the application
    - Use object-oriented design and WTForms
    - Learn the difference between `GET` and `POST` HTTP methods; use them properly
4. Deployment, error handling, and testing
    - Use a public hosting service to deploy the application
    - Learn to use defensive programming to handle errors in the application
    - Use *pytest* and *playwright* to test the application
5. Permanent storage
    - Use a database to store data
    - Learn common methods of data modeling and querying
    - Use SQL for basic operations on data (Create, Read, Update, and Delete)
    - Learn about object-relational mappers
6. Authentication and authorization
    - Use best development practices to structure the application
    - Add user management functionality to the application

### Research project

Final project is open-ended but students are expected to build an application with a consistent user interface, persistent data storage, and custom logic.
